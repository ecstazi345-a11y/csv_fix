# Регламент синхронизации данных (P1.1)

> **Зачем:** KPI на витрине Streamlit актуальны только после синка Airtable → Supabase.  
> **Когда смотреть качество данных:** страница **«Качество данных»** (`pages/06_Качество_данных.py`).

---

## Когда запускать синк

| Ситуация | Действие |
|----------|----------|
| Утро перед совещанием / обзором витрины | Запустить полный синк |
| После ввода сменных отчётов мастерами (конец смены) | Запустить полный синк |
| После правок в Monthly Passport Plan | Запустить синк паспорта (или полный) |
| Перед экспортом / скриншотом для заказчика | Запустить полный синк |
| «Цифры не сходятся» с Airtable | Синк + проверка **Качество данных** |

**Не обязательно** синкать каждый час — достаточно **2–4 раза в рабочий день** + по событию.

---

## Как запустить (Windows)

**Вариант 1 — один клик:**

```
C:\csv_fix\ОБНОВИТЬ_ВСЕ_ДАННЫЕ.bat
```

**Вариант 2 — PowerShell:**

```powershell
cd C:\csv_fix
.\.venv\Scripts\Activate.ps1
python update_all_sync.py
```

**Порядок скриптов внутри `update_all_sync.py` (4 синка):**

1. `daily_progress_sync_upsert.py` — факт с площадки (Daily Progress)  
2. `boq_sync_upsert.py` — справочник BOQ  
3. `monthly_passport_sync_airtable.py` — план месяца (Monthly Passport)  
4. `monthly_labor_summary_sync_upsert.py` — звенья / трудозатраты (Airtable `Crew_Register`, view `SUPABASE_MONTHLY_LABOR_SUMMARY` → Supabase `monthly_labor_summary`)  

---

## После синка — проверка

1. Открыть Streamlit: http://localhost:8501  
2. **Главная** / **Исполнение** — KPI обновились  
3. **Качество данных** — нет ли новых WRONG_FACILITY / WRONG_DISCIPLINE  

Если ошибок много — сначала исправить ввод в Airtable, затем снова синк.

---

## Если синк упал

| Симптом | Что проверить |
|---------|----------------|
| Ошибка Airtable API | `.env`: `AIRTABLE_TOKEN`, `AIRTABLE_BASE_ID`, ID таблиц |
| Ошибка Supabase | `.env`: `SUPABASE_URL`, `SUPABASE_KEY` |
| Частичные данные | Запустить синк повторно; смотреть вывод в консоли |

Не править данные в Supabase вручную — источник ввода **Airtable**.

## Ответственность

| Роль | Действие |
|------|----------|
| Мастера / планировщики | Корректный ввод Project, Facility, IWP, System, Discipline |
| Диспетчер / ПТО | Синк по регламенту, контроль **Качество данных** |
| Руководитель | Решения по отклонениям на **Исполнение**, не до синка |
